一:添加多渠道的变量
 应用：每个渠道使用 ”同一位置 “的变量不同 
1  productFlavors {
	
            //注意是‘’ 然后逗号分隔
            buildConfigField 'String', 'authorities_id','"com.szqd.sznotes.fileprovider"'
        }
	
	}
2 编译版本的时候会在 E:\code\android\sznotes\app\build\generated\source\buildConfig\多渠道名\debug\com\szqd\sznotes\BuildConfig.java生成	
package com.szqd.sznotes;

public final class BuildConfig {
  // Fields from product flavor: a_default
  public static final String authorities_id = "com.szqd.sznotes.fileprovider";
}

3 代码中使用
  因为 BuildConfig类的包名都是一样的
  所以 只用  BuildConfig.authorities_id, 调用即可

三:替换使用的资源或代码
     切换到project 标签下 在app-》src上右键 -》new->destrory- 
	 Res （会自动替换为渠道工程里的）
	  建立了res 后就可以根据具体的替换来添加文件，注意名字都要一样
	  value-》string 文件里的内容 颗粒到单个string变量
	 java（不会替换 你需要删除main工程下对应的java文件 并创建其他渠道工程文件）
	  修改对应代码也一样 但是要保证包名包含的文件一样 都用 destrory 创建就行 我靠好像不能自动替换
